# Apple Notes Integration

## MCP Server: apple-notes

Используется пакет `mcp-apple-notes` через `bunx mcp-apple-notes`.

---

## Available Tools

### list_notes — список заметок в папке

```
folder: "Проекты рабочие"   # имя папки
limit: 20                    # максимум заметок (default: 20)
```

### get_note_content — содержимое заметки

```
note_name: "Название заметки"  # ОБЯЗАТЕЛЬНО
folder: "Проекты рабочие"      # опционально (для уточнения)
```

### add_note — создать заметку

```
name: "Название заметки"       # ОБЯЗАТЕЛЬНО
content: "Текст заметки"       # ОБЯЗАТЕЛЬНО (поддерживает markdown)
folder: "Проекты рабочие"      # ОБЯЗАТЕЛЬНО (default: "Notes")
```

### update_note_content — обновить заметку

```
note_name: "Название заметки"  # ОБЯЗАТЕЛЬНО
new_content: "Новый текст"     # ОБЯЗАТЕЛЬНО
folder: "Проекты рабочие"      # опционально
```

## Folder Mapping для GTD

| Тип контента | Папка Notes | Когда использовать |
|-------------|-------------|-------------------|
| Описание проекта, outcome, шаги | Проекты рабочие | При создании PROJECT |
| AI-инструменты, промпты | AI | AI-related reference |
| Цели, планы, видение | Планирование | Стратегическое |
| Продуктивность, системы | Продуктивность | GTD-материалы |
| Личное развитие, книги | Личностный рост | Инсайты, обучение |
| Некоммерческий фонд | Фонд | Задачи фонда |
| Видеоконтент | Канал YouTube | YouTube-идеи |
| Здоровье, анализы | Здоровье | Медицинское |
| GTD-база | GTD | Справочные GTD-материалы |

## Project Note Format

При создании описания проекта в Notes использовать структуру:

```markdown
## [Название проекта]

**Desired Outcome:** [Как выглядит успех?]
**Why:** [Зачем это важно?]
**Due:** [Когда нужно завершить?]

### Next Actions
- [ ] [Первое конкретное действие]
- [ ] [Второе действие]

### Notes
[Контекст, ссылки, детали]
```

---

## Reference Note Format

При сохранении справочного материала:

```markdown
**Источник:** [откуда]
**Дата:** [YYYY-MM-DD]
**Теги:** #тег1 #тег2

[Основной контент]

**Применение:** [Как использовать этот материал]
```

---

## Anti-Patterns

- ❌ Не дублировать задачи из Reminders в Notes
- ❌ Не использовать Notes как inbox (для этого есть Reminders "inbox")
- ❌ Не создавать заметку без указания папки
- ✅ Notes = Reference + Project descriptions
- ✅ Reminders = Actions + Tasks
